Introduction

ToolKit is a nice client side JavaScript library that offers something a little different that you'd expect to see in most JavaScript libraries. ToolKit doesn't offer a great deal in the way that a tool such as jQuery, but rather more advance tools, an example being the multithreading wrapper that's included. ToolKit comes with a few very neat, fast and reasonably powerful and stable features, if you like you can customise which features you wish to include by having a look at the following components here.

Why use ToolKit?

This is a simple question, and it deserves a simple answer, ToolKit is simply made by developer(s) whom are passionate about getting the best performance:bloat:power:featureful ratio possible.

Targeting DOM Elements

Out of the box, the ultimate aim of ToolKit is to provide a front end developer with some tools that a front end developer may need, without the cost of overloading the front end with bloated code nor does it include a great deal of complexity. This is a part of the reason as to why ToolKit doesn't come with an endless supply of tools, such as some of the many DOM manipulation tools that you'd find in something like jQuery, as an example.

If you wish to stay away from jQuery, then ToolKit does include a simple feature that allows you to target DOM elements, however the style is ever so slightly different to prevent any issues with those whom wish to use jQuery. If you wish to target DOM elements in a similar fashion to jQuery's DOM targeting tool, you can do it like so:

jQuery's implementation: 
$
ToolKit's implementation: 
ToolKit.$e

AJAX

Furthermore, if you wish to use AJAX, you can do so with ToolKit also. One neat feature about ToolKit's implementation of AJAX is that it will automatically set the 'no-sniff' header, just to help ensure that your front end application is secure. However, there are a few subtle differences when comparing the jQuery implementation of AJAX to the ToolKit implementation. For one they take parameters that are slightly different, however if you're familiar with jQuery's AJAX, this shouldn't be anything new.

jQuery's implementation: 
$.ajax
ToolKit's implementation: 
ToolKit.AJAX
